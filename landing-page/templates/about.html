What {{ app_name }} does

Point it at a folder, archive (.7z / .zip) or a single .sql file. It scans every file in natural-sort order, collects every DELETE FROM … statement into a reviewable delete.sql, then generates timestamped BACKUP.sql (CREATE TABLE AS SELECT snapshots) and REVERT.sql (DELETE + conditional INSERT-from-backup) you can run later if rollback is needed.

Security & data storage

User accounts & passwords
Stored in a local SQLite database at {{ db_path }} (table users). Passwords are never stored in plaintext — they're hashed with PBKDF2-SHA256 (Werkzeug generate_password_hash, per-user salt, 600k iterations). You can inspect the DB with sqlite3 data/app.db ".schema users".
Sessions
Signed Flask cookies — signature is keyed by FLASK_SECRET_KEY from your .env. Rotate the key to invalidate all existing sessions.
Password-reset tokens
Single-use, hashed before storage (same PBKDF2), default TTL 60 minutes. The cleartext token only exists in the email (or the server log if SMTP isn't configured).
API tokens
Only the prefix is stored in the clear; the full token is hashed. Lost tokens cannot be recovered — revoke and re-issue.
Uploads & generated files
Each job lives in its own UUID folder under {{ upload_root }}. Background sweep removes unclaimed folders after CLEANUP_MINUTES and retired jobs after JOB_RETENTION_DAYS.
